index.html — how the app boots and fits together

One page showing what index.html loads, the order it runs in, which file imports which, and what each file is responsible for. All app code lives in zmath/.

Note: this index.html uses real ES modules (import / export) resolved by a small in-page loader — not the old "Babel script tags + window.X = X globals" setup. Files share code by importing it directly, not through window.

B Boot sequence what index.html does, top to bottom

  1. Load the one stylesheet
    zmath/styles.css — the brain owns the full CSS list; the shell links only this file.
  2. Register the import map
    Points the bare names react and react-dom at esm.sh — the same specifiers the Vite build uses, so source files are identical for both readers.
  3. Load Babel standalone
    From unpkg (pinned). Used by the loader to compile JSX in the browser.
  4. Run the graph loader on ./zmath/main.jsx
    Walks the import graph: fetches each relative module, Babel-transforms its JSX, rewrites relative imports to blob URLs, then imports the entry. Bare names fall through to the import map.
  5. main.jsx mounts the app
    Sets the document title, wraps in an error boundary, and createRoot(#root).render(<App/>).
