What everything is, in plain words

Only three names to know. Read them once and refer back whenever a word confuses you.

VS Code a free app from Microsoft for opening and running projects on your own computer. It has a file list, a big area in the middle, and a text box at the bottom called the terminal. It's where you'll do everything.
Node a free helper that VS Code uses to actually start the project. You install it once and never touch it directly — the project just needs it present in the background.
The studio where I (your assistant) build the project. When you ask me for "the zip," I hand you a packaged, up-to-date copy of the whole project to run.

Two commands you'll type

In the terminal you'll type two short lines. Here's all they do:

  • npm install = fetch the extra pieces the project needs. Slow the first time (a few minutes), then you rarely run it again.
  • npm run dev = start the project. It gives you a web address to open in your browser.
The one rule: the project has two folders that sit next to each other — AlphabetMath - App and AlphabetMath - Mobile. One relies on the other, so keep the whole project folder together, exactly as it comes out of the zip. Following the steps below does this for you.

Set up your computer (do this once)

Two free installs, then you get the project files. Set aside about 15 minutes. You never repeat this section.

  1. Install VS Code. Go to code.visualstudio.com, click the big download button, and install it like any app. Open it once so you know it's there.
  2. Install Node. Go to nodejs.org and download the button marked LTS (the recommended one). Install it, clicking Next / Agree through every screen — the defaults are correct. When it finishes, Node is running quietly in the background; there's nothing to open.
  3. Ask me for the zip. Type "give me the zip." I hand you a single packaged file. Save it somewhere easy to find, like your Desktop or Documents.
  4. Unzip it. Right-click the file → Extract All (Windows) or double-click it (Mac). You now have a project folder full of files. Move it somewhere you'll remember — for example a folder called AlphabetMath on your Desktop.
    • Open the folder to check. It should contain AlphabetMath - App, AlphabetMath - Mobile, and a few other files side by side. If instead you see just one folder inside, go into it — the real project is one level down.
That's the whole setup. From now on you only ever do Section 3.

Open and run the project

This is the routine you'll repeat. The first run takes a few minutes (it fetches its pieces); after that it starts in seconds.

  1. Open the project in VS Code. Open VS Code, then click File → Open Folder and choose your AlphabetMath project folder (the one that holds the two AlphabetMath - folders). Its files appear down the left side.
    • If a bar pops up asking "Do you trust the authors?", click Yes, I trust.
  2. Open the terminal. In the top menu click Terminal → New Terminal. A text box opens along the bottom. This is where the next lines go.
  3. Move into the App folder. Type this line and press Enter: 
    cd "AlphabetMath - App"
    • The quote marks matter — the folder name has spaces in it.
  4. First time only — fetch the pieces. Type this and press Enter, then wait for it to finish (a few minutes; lots of text is normal): 
    npm install
    • You only do this the very first time, and again whenever I tell you a new zip needs it. Otherwise skip straight to the next step.
  5. Start it. Type this and press Enter: 
    npm run dev
    • After a moment it prints a web address like http://localhost:5173/. Hold Ctrl (or Cmd on Mac) and click it, or copy it into your browser. The AlphabetMath home screen loads.
  6. When you're done. Click inside the terminal and press Ctrl + C to stop it. You can close VS Code entirely. Next time, just reopen VS Code (it remembers the folder) and run steps 2, 3, and 5 — that's it.
The short version, every time after setup: open VS Code → Terminal → New Terminal → cd "AlphabetMath - App" → npm run dev → click the link. Under a minute.

Get the latest updates from me

While I'm still building the project, every update is the same short swap:

1. Ask me for a fresh zip.
2. Unzip it, replacing your old project folder with the new one (say "yes / replace" if it asks).
3. In VS Code, run it again (Section 3, steps 2, 3, 5).
4. If I tell you the update changed its pieces, also run npm install (Section 3, step 4) before starting.

That's the whole cycle. When in doubt, running npm install once before npm run dev never hurts — it just makes sure everything's present.

If something goes wrong

The few snags that actually come up, with the fix for each.

  • “npm is not recognized / command not found.” — Node isn't installed, or VS Code opened before it was. Close VS Code completely, reopen it, and try again. If it still says this, reinstall Node (Section 2, step 2).
  • “It can't find the folder after cd.” — Check the quote marks and the exact spacing: cd "AlphabetMath - App". You must have the project folder open first (Section 3, step 1).
  • “The terminal printed a lot of red text.” — If a web address still appeared and the app opened, ignore it. If nothing opened, copy the red text and send it to me.
  • “The link says the page can't be reached.” — The project probably isn't running. Make sure npm run dev is still going in the terminal (don't close that window while you're using the app).
  • “It opened but looks broken or half-empty.” — You may have opened the wrong folder, or skipped npm install. Stop it (Ctrl + C), run npm install, then npm run dev again.
  • “Unzipping asked about replacing files.” — Choose replace / overwrite. The new copy is the newer version — that's what you want.
