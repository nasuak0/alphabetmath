The iPad Port Guide

AlphabetMath · getting the project into your Codespace, with pictures

What this is

Your Codespace works — but the project files never made it to GitHub, so it only shows a README. This guide fixes that, entirely from the iPad, in six short acts: know the screen → get the zip → drop it in → save to GitHub → run the app → publish it as a live website. Every act comes with a picture of what you'll see. The pictures are simplified sketches — colors and exact icons may differ slightly on your screen, but the positions are right.

Act 1 — Get the zip onto your iPad

  1. Find the download card in our chat. Right under this guide in the chat there's a card named alphabetmath.zip. Tap it.
  2. Save it to Files. Safari shows a download arrow (top-right). Tap it → tap alphabetmath.zip. If asked where, choose On My iPad → Downloads. That's it — the project is now on your iPad.

Act 2 — Drop the zip into the Codespace

Two ways; use whichever feels easier.

Way A — touch and hold (recommended). In the Explorer (area 3), touch and hold on the empty space below README.md. A menu pops up:

Touch-and-hold in the empty Explorer space → tap Upload…

Tap Upload…, and in the file picker choose alphabetmath.zip from Downloads. A little progress bar runs; then alphabetmath.zip appears in your file list under README.md.

Way B — drag it. Open the Files app in Split View next to Safari, then drag alphabetmath.zip out of Files and drop it onto the Explorer file list.

Act 3 — Unpack it (one paste)

  1. Open the terminal. ☰ menu → Terminal → New Terminal (or tap the TERMINAL tab if the bottom panel is already open).
  2. Copy this whole block, paste it into the terminal, press Enter. It unpacks the zip into place and deletes the zip. It looks scary; it is one command. 
    unzip -q alphabetmath.zip -d /tmp/p && shopt -s dotglob && { [ -d /tmp/p/alphabetmath ] && mv /tmp/p/alphabetmath/* . || mv /tmp/p/* . ; } && rm -rf /tmp/p alphabetmath.zip && ls -A
  3. Check the result. The terminal prints the new file list — you should see AlphabetMath - App, AlphabetMath - Mobile, interactives and more. The Explorer on the left fills up too. If it did — the hard part is over.

Act 4 — Save it to GitHub

The files are in the Codespace, but not yet saved to GitHub itself. Use the visual way:

The Source Control panel: message box on top, blue button under it

  1. Open Source Control. Tap the 3rd icon in the left strip (branching lines, probably showing a number bubble).
  2. Type a name in the message box: port project.
  3. Tap the blue Commit button. If a popup asks about "staging changes", tap Yes (or Always).
  4. Tap the button again — it now says "Sync Changes". That pushes everything up to GitHub. When the file list clears out, it's done and permanent: even if this Codespace is deleted, the project is safe on GitHub.

Act 5 — Run the app

  1. In the terminal, paste this and press Enter: 
    cd "AlphabetMath - App" && npm install && npm run dev
    It installs the app's parts (a minute or two, first time only), then starts it.
  2. Watch for the popup, bottom-right:
    Your application running on port 5173 is available.
    Open in Browser

    Tap Open in Browser

  3. One quirk: if the new tab shows "404 Not Found", add alphabetmath/ to the end of the address and reload — the app lives at …github.dev/alphabetmath/. That's the app, live.
  4. Missed the popup? Tap the PORTS tab in the bottom panel, find the row for 5173, touch-and-hold it → Open in Browser.
  5. To stop: tap in the terminal and press Ctrl + C (on-screen keyboard: tap the ⌃ key, then C). To leave: just close the tab.
Every time after this: github.com → your repo → green Code button → Codespaces tab → tap your Codespace. Then only step 1 above (and npm install finishes instantly once it's been done).

Act 6 — Make it a live website

Every push (Act 4) already carries an instruction telling GitHub to rebuild a public website. You only have to tell GitHub to obey it — once. This act happens on github.com, not in the Codespace.

  1. Go to your repository page. In Safari: github.com/YOUR-USERNAME/alphabetmath. Tap the Settings tab in the row across the top (on a narrow screen it hides behind a ⋯ at the end of that row).

Bonus — Claude Code in the terminal

Claude Code doesn't run on the iPad itself — it runs inside the Codespace terminal, and the iPad is the window into it. Once the project is pushed (Act 4):

  1. Install it (once):
    npm install -g @anthropic-ai/claude-code
  2. Start it from the project folder:
    cd /workspaces/alphabetmath && claude
  3. Sign in. The first run prints a login link — tap it, sign in with your Anthropic account, come back to the terminal. After that, you just talk to it.
One rule to keep: the moment you start editing in the Codespace (with Claude Code or by hand), the design studio must stop editing its copy — one editor at a time, or the two copies drift apart. Running the app is always safe.

If something goes wrong

  • "unzip: cannot find alphabetmath.zip." — Safari may have saved it as alphabetmath (1).zip. Type ls to see the real name, then touch-and-hold the file in the Explorer → Rename → alphabetmath.zip, and retry.
  • Can't paste into the terminal. — Tap once inside the terminal first, then touch-and-hold → Paste. Safari may ask permission to read the clipboard — allow it.
  • The Commit button is greyed out. — The message box is empty. Type port project first.
  • Red text after npm run dev. — If the app still opened, ignore it. If nothing opened, copy the red text and send it to me.
  • The browser tab shows an old or blank page. — Reload once; if still wrong, check the address ends in /alphabetmath/.
  • The public link shows 404 / not found. — One of three things: the Pages source isn't set to GitHub Actions (Act 6, step 2); the build hasn't finished (green check in the Actions tab yet?); or the address is missing the final /alphabetmath/.
  • The Codespace itself is stuck. — Close the tab, reopen from github.com → Code → Codespaces. Your pushed files are safe on GitHub no matter what.
